# Albion Online Craft Calculator Foundation ve Sprint Akışı

## Context

- İş problemi:
  Albion Online için aşırı detaylı, güvenilir ve genişleyebilir bir craft calculator geliştirmek istiyoruz.
- Mevcut durum:
  Repo içinde CrewAI orchestration, request/run/report sistemi, probe/scoring sistemi ve craft calculator için ilk domain foundation başlatıldı.
- İlgili dosyalar:
  - `src/albion_factory/craftcalc/`
  - `knowledge/craft_calculator_sprint_01.md`
  - `src/albion_factory/config/factory_agents.yaml`
  - `src/albion_factory/config/factory_tasks.yaml`
  - `src/albion_factory/factory_crew.py`
  - `src/albion_factory/factory_flow.py`

## Goal

- Beklenen sonuç:
  Craft calculator için request-temelli, ajanlara dağıtılmış, sprint bazlı uygulama akışı başlasın.
- Başarı kriteri:
  1. Lead sprint scope ve görev dağılımını net yazsın.
  2. Researcher veri/model gereksinimlerini netleştirsin.
  3. Coder foundation iskeletini küçük güvenli adımlarla uygulasın.
  4. Tester evidence üretsin.
  5. Validator PASS/PARTIAL/FAIL ve commit_ready kararı versin.

## Constraints

- Dokunulmayacak alanlar:
  - mevcut request/run/report append-only mantığı bozulmayacak
  - scope dışı büyük refactor yapılmayacak
- Riskli noktalar:
  - recipe/price/rule veri modeli yanlış soyutlanırsa ileride tüm sistem zarar görür
  - canlı veri ile statik veri ayrımı bulanıklaşmamalı
  - agentler soyut ve anlamsız cevap vermemeli
- Kullanılabilecek kaynaklar:
  - AODP / resmi kaynaklar
  - repo içi knowledge ve reference dosyaları
  - mevcut craftcalc foundation

## Output Expectation

- İstenen çıktı formatı:
  - Lead: sprint scope + success criteria + next step
  - Researcher: data source inventory + domain findings + constraints
  - Coder: target files + planned/applied changes + remaining risks
  - Tester: commands + evidence + findings + verdict
  - Validator: verdict + evidence used + required fixes + commit ready
- Test beklentisi:
  - compile / import smoke test
  - craftcalc modelleri import doğrulaması
- Commit beklentisi:
  - validator yeterli evidence olmadan commit_ready dememeli

## Planned Phases

1. Domain foundation
2. Recipe resolver
3. Price resolver
4. Profitability engine
5. Comparison engine
6. Validation / probe / scoring

## Immediate Sprint Task

Bu request için sadece şu sprint uygulanacak:
- craft calculator foundation'ın request-temelli ilk sprint akışını başlat
- ajanlar kendi rollerine göre çıktı üretsin
- küçük, güvenli ve commit-ready bir temel adım teslim edilsin
